# Recipes

Workflows that **worked**. When we figure out a sequence of steps that produces good results, we save it here so we can repeat it without re-figuring it out.

## What belongs in a recipe

- A specific outcome (not "make a video" — more like "sync captions to TTS narration with bouncy per-word entry")
- The exact steps in order
- The key configs (resolution, FPS, voice, model versions, etc.)
- What to watch out for ("if this happens, do that")
- A link to a finished example in `compositions/`

## What does NOT belong

- Prompts (those go in `prompts/`)
- General concepts or theory (those go in `memory.md`)
- Experimental stuff that worked once (wait until you've used it 2–3 times)

## Naming convention

`<outcome>.md` — lowercase, hyphen-separated. Examples:
- `sync-tts-to-captions.md`
- `bar-chart-race-with-photos.md`
- `lower-third-with-avatar-cutout.md`

## Template

Use `_template.md` as a starting point when you add a new recipe.
