# Competitor Video Analysis

## When to use
You found a competitor's video that's performing well. You want to understand *why* so you can borrow the patterns (not copy the content) and apply them to your own work.

## Prompt

```
Analyze this competitor video and break down what's working, so we can apply the same playbook to our own content.

Video: [URL or file path]
Competitor: [name]
Why I picked this one: ["it has X views" / "it ranks for Y" / "they post a lot of these"]

What I want from the breakdown:
1. **Hook analysis** (first 3 seconds)
   - What's the opening visual?
   - What's the opening line / on-screen text?
   - Why does it stop the scroll?

2. **Structure** (scene-by-scene)
   - List every scene with a timestamp, what happens, and why it's there
   - Note transitions between scenes

3. **Visual style**
   - Color palette (eyedrop the key colors)
   - Typography (best guess at the fonts, or just describe — serif/sans/display, weight, size relative to screen)
   - Motion language: snappy? smooth? sliding? popping? camera moves?
   - Use of footage vs animation vs both

4. **Audio**
   - Voiceover (human / AI / accent / pace / energy)
   - Music (mood / instrumentation / volume relative to voice)
   - Sound design (transitions, UI sounds, emphasis hits)

5. **Caption treatment**
   - Position on screen
   - Style (color, weight, animation)
   - Sync to voice (per word / per phrase / static)

6. **The playbook**
   - Distill the WHOLE thing into a 5-line repeatable recipe
   - Specifically: "if I were making this exact type of video, my brief would be:"

7. **What I should steal**
   - 3 specific techniques worth borrowing
   - 1 thing they did that I should NOT do (overdone, confusing, etc.)

Save the breakdown to scripts/competitor-analyses/[competitor-name]-[YYYY-MM-DD].md

If you have access to the video-analyzer skill, USE IT — actually watch the video, don't guess.
```

## Notes
- The goal is to extract the *pattern*, not the script. Same hook structure ≠ stolen content.
- Save the analyses — they become a competitive intel library over time
- If multiple competitors run the same pattern, that's a confirmed best practice for the niche
